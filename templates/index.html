Tokenization

Breaking text into its smallest meaningful units — tokens

What is Tokenization?

Tokenization is the process of splitting text into individual pieces called tokens. Tokens are usually words, punctuation marks, or numbers — the atomic units that NLP models process.

1
spaCy's tokenizer handles contractions: don't → do + n't
2
Punctuation is split from words automatically
3
Every token has a unique index (position) in the text
4
The first step in every spaCy pipeline
Token types: word punct digit stop word

🔤 Try Tokenization

🔤
Tokens will appear here as colorful chips
🐍 Tokenization with spaCy 
import spacy
nlp = spacy.load("en_core_web_sm")

doc = nlp("Apple is looking at buying U.K. startup for $1 billion!")

for token in doc:
    print(f"{token.text:15} is_stop={token.is_stop} is_punct={token.is_punct}")

# Output:
# Apple          is_stop=False is_punct=False
# is             is_stop=True  is_punct=False
# looking        is_stop=False is_punct=False
# ...            ...
# !              is_stop=False is_punct=True

# Token attributes:
token = doc[0]
print(token.text)    # 'Apple'
print(token.idx)     # character offset
print(token.is_alpha)# True
print(token.is_stop) # False

Sentence Segmentation

Detecting sentence boundaries in continuous text

What is Sentence Segmentation?

Sentence segmentation (or sentence boundary detection) identifies where one sentence ends and another begins. This is harder than it sounds — a period after "Dr." doesn't end a sentence!

1
Uses the dependency parser to find sentence roots
2
Handles abbreviations like U.S.A., Dr., etc.
3
Essential for summarization and translation tasks
4
Accessed via doc.sents

📝 Try Sentence Segmentation

📝
Detected sentences will appear color-coded
🐍 Sentence Segmentation with spaCy 
import spacy
nlp = spacy.load("en_core_web_sm")

text = "NLP is fascinating. Dr. Smith agrees! Do you?"
doc = nlp(text)

# Iterate over detected sentences
for i, sent in enumerate(doc.sents):
    print(f"Sentence {i+1}: {sent.text}")
    print(f"  Tokens: {len(sent)}")

# Convert to list
sentences = list(doc.sents)
print(f"Total sentences: {len(sentences)}")

Part-of-Speech Tagging

Labeling every word with its grammatical role

What is POS Tagging?

POS tagging assigns a grammatical category — noun, verb, adjective, etc. — to each token. It's the foundation for understanding sentence structure.

NOUN — person, place, thing
VERB — action, state
ADJ — describes noun
ADV — modifies verb/adj
PROPN — proper noun
CCONJ — and, but, or
PRON — he, she, it
DET — the, a, an

🏷️ Try POS Tagging

🏷️
Words will appear with color-coded POS tags below
🐍 POS Tagging with spaCy 
import spacy
nlp = spacy.load("en_core_web_sm")

doc = nlp("The ancient dragon soared gracefully.")

for token in doc:
    print(f"{token.text:12} POS={token.pos_:8} TAG={token.tag_}")

# token.pos_  — coarse-grained (NOUN, VERB, ADJ…)
# token.tag_  — fine-grained  (NN, VBD, JJ…)

# Get human-readable explanation
print(spacy.explain("VBD"))  # verb, past tense
print(spacy.explain("NOUN")) # noun

# Filter by POS
nouns = [t.text for t in doc if t.pos_ == "NOUN"]
verbs = [t.text for t in doc if t.pos_ == "VERB"]

Named Entity Recognition

Identifying and classifying real-world entities in text

What is NER?

Named Entity Recognition (NER) locates and classifies named entities — real-world objects like people, organizations, and locations — in unstructured text.

👤
PERSON — People, characters
🏢
ORG — Companies, agencies, institutions
🌍
GPE — Countries, cities, states
📅
DATE — Absolute or relative dates
💰
MONEY — Monetary values
📍
LOC — Non-GPE locations

🎯 Try Named Entity Recognition

🎯
Entities will be highlighted with color-coded labels
🐍 Named Entity Recognition with spaCy 
import spacy
nlp = spacy.load("en_core_web_sm")

doc = nlp("Elon Musk founded Tesla in California in 2003.")

# Iterate over named entities
for ent in doc.ents:
    print(f"{ent.text:20} {ent.label_:10} {spacy.explain(ent.label_)}")

# Elon Musk            PERSON     People, including fictional
# Tesla                ORG        Companies, agencies, institutions
# California           GPE        Countries, cities, states
# 2003                 DATE       Absolute or relative dates

# Visualize in browser (Jupyter / Flask)
from spacy import displacy
displacy.render(doc, style="ent")  # → HTML

# Entity properties
ent = doc.ents[0]
print(ent.start_char, ent.end_char)  # character offsets

Dependency Parsing

Revealing the grammatical structure and relationships between words

What is Dependency Parsing?

Dependency parsing analyzes the grammatical structure of a sentence, establishing relationships between "head" words and words that depend on them.

ROOT
The main verb — the root of the sentence
nsubj
Nominal subject — who is doing the action
dobj
Direct object — what receives the action
amod
Adjectival modifier — adjective modifying a noun
prep
Prepositional modifier (on, in, at…)
💡 Try a short sentence like "The cat sat on the mat." for the clearest result.

🌳 Try Dependency Parsing

🌳
Dependency relations shown as an annotated table
🐍 Dependency Parsing with spaCy 
import spacy
nlp = spacy.load("en_core_web_sm")

doc = nlp("The cat sat on the mat.")

for token in doc:
    print(f"{token.text:8} dep={token.dep_:10} head={token.head.text}")

# Find the root verb
root = [t for t in doc if t.dep_ == "ROOT"][0]
print(f"Root: {root.text}")

# Subject and object
subject = [t for t in doc if t.dep_ == "nsubj"]
dobj    = [t for t in doc if t.dep_ == "dobj"]

# Visualize the tree
from spacy import displacy
displacy.render(doc, style="dep")

Lemmatization

Reducing words to their base dictionary form

What is Lemmatization?

Lemmatization reduces inflected words to their lemma — the base dictionary form. Unlike stemming, it produces real words by considering the word's meaning and context.

running → run
better → good
children → child
was → be

📚 Try Lemmatization

📚
Changed words highlighted in green
🐍 Lemmatization with spaCy 
import spacy
nlp = spacy.load("en_core_web_sm")

doc = nlp("The children were running and the dogs barked.")

for token in doc:
    if not token.is_punct and not token.is_space:
        print(f"{token.text:15} → {token.lemma_}")

# children → child
# were     → be
# running  → run
# dogs     → dog
# barked   → bark

# Lemmatization vs Stemming:
# Stemming:       running → run  (but also: better → bet ❌)
# Lemmatization:  better  → good (context-aware ✓)

Stop Words

Filtering out common words that carry little meaning

What are Stop Words?

Stop words are very common words — the, is, at, which, on — that appear frequently but carry little semantic meaning. Removing them reduces noise and improves NLP model efficiency.

✓
spaCy has 326 English stop words built-in
✓
Improves TF-IDF, topic modeling, search
✓
Can add/remove custom stop words
⚠
Not always helpful — sentiment analysis needs "not"!
Common stop words: the, a, an, is, are, was, were, be, been, have, has, had, do, does, did, will, would, could, should, and, or, but, in, on, at, to, of, for…

🚫 Try Stop Word Removal

🚫
Stop words shown crossed out, content words highlighted
🐍 Stop Words with spaCy 
import spacy
nlp = spacy.load("en_core_web_sm")

doc = nlp("NLP is a fascinating field of computer science.")

# Filter out stop words
content = [t.text for t in doc
           if not t.is_stop and not t.is_punct]
print(content)  # ['NLP', 'fascinating', 'field', 'computer', 'science']

# Check a word
print(nlp.vocab["the"].is_stop)  # True
print(nlp.vocab["nlp"].is_stop)  # False

# Customize: add/remove stop words
nlp.Defaults.stop_words.add("nlp")
nlp.vocab["nlp"].is_stop = True

nlp.Defaults.stop_words.discard("not")
nlp.vocab["not"].is_stop = False

Text Similarity

Measuring how semantically close two pieces of text are

What is Text Similarity?

spaCy measures similarity using word vectors — numerical representations of words in a high-dimensional space where similar words are close together.

📐
Uses cosine similarity between vector representations
🔢
Returns a score from 0.0 (different) to 1.0 (identical)
🧠
Captures semantic meaning, not just exact word matches
🌐
Used in duplicate detection, recommendation, search
⚠ The small model (en_core_web_sm) has limited vectors. For production use en_core_web_md or en_core_web_lg.

🔗 Try Text Similarity

🔗
Similarity score will appear as a percentage meter
🐍 Text Similarity with spaCy 
import spacy
nlp = spacy.load("en_core_web_md")  # md/lg has better vectors

doc1 = nlp("I love playing basketball.")
doc2 = nlp("Playing sports is my hobby.")

# Document-level similarity
print(doc1.similarity(doc2))  # e.g. 0.87

# Token-level similarity
king  = nlp("king")[0]
queen = nlp("queen")[0]
man   = nlp("man")[0]
print(king.similarity(queen))  # high
print(king.similarity(man))    # moderate

# Word vectors (300-dim by default)
print(king.vector.shape)       # (300,)
print(king.has_vector)         # True
